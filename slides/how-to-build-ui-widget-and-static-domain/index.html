怎么解决？

UI多态化

何为UI多态化？它可以做什么？

  • 以某一种基础UI为祖先，衍生出不同维度上变化。
  • 不同维度有：色系、尺寸、状态、多终端适配、交互特性等方面
  • 衍生方案采用继承式（堆叠式）。

动效的实现方案

  • 渐进增强。
  • 依赖CSS3，禁用JS动画。
  • 不滥用，在需要的场景下使用合适的动效。
  • 不夸张，拒绝大开大合的动效。（个人观点）

此阶段中，还有一件事我们要解决，就是UI的多终端的支持。

  • 什么是响应式？（Responsive design，RWD）
  • 什么是自适应？（Adaptive design，AWD）
  • RWD：倾向基于Media Queries、Content-Based Breakpoint来改变页面适配不同的分辨率尺寸。
  • AWD：倾向针对几种特定的分辨率尺寸做定制。可能会有好几套页面。

场景举例

多终端适配支持，我们会从以下几个方面去考量

  • 栅格系统的适配
  • UI尺寸的适配
  • UI特效的适配
  • UI交互的适配

在组件化方案产出之前，多终端的适配不会过多的去考虑特效和交互

第三阶段
组件化

什么是UI？什么组件？

  • UI是一套视觉规范和风格。
  • 组件是对这套视觉规范和风格的实现。

什么是组件化？

  • 系统化，对现有视觉规范和风格的统一实现，自成体系。
  • 通用性，与具体业务隔离。
  • 易用性，在不同的业务场景下通过入参可以定制差异化的具体表现甚至是交互逻辑。
  • 易拓展、易维护、等等
  • Sencha ExtJS
  • jQuery UI
  • ANT DESIGN

我们的组件化计划，Nyx-Components

组件化主要做哪些事？

  • 组件化实现方式的探索。
  • 组件化的派生体系。
  • 构建基础组件及组件分类。
  • 构建复杂组件及交互实现。
